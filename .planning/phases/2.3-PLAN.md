# Phase 2.3: Implement NextAuth.js with Credentials Provider

**Phase:** 2
**Plan:** 2.3
**Status:** Ready to Execute

---

## Objective

Set up NextAuth.js authentication with email/password credentials.

---

## Tasks

### 1. Install NextAuth.js
- [ ] Run `npm install next-auth@beta` (for Next.js 16 compatibility)
- [ ] Run `npm install bcryptjs` for password hashing
- [ ] Run `npm install -D @types/bcryptjs` for TypeScript types

### 2. Create Auth Config
- [ ] Create `src/lib/auth.ts` with NextAuth configuration
- [ ] Configure Credentials provider
- [ ] Set up JWT strategy
- [ ] Configure session callbacks

### 3. Implement Authorization Logic
- [ ] Create `src/lib/auth-queries.ts` for database queries
- [ ] Implement `getUserByEmail()` function
- [ ] Implement password verification with bcrypt
- [ ] Create user session object (without password)

### 4. Create Auth API Route
- [ ] Create `src/app/api/auth/[...nextauth]/route.ts`
- [ ] Export GET and POST handlers from NextAuth
- [ ] Configure secret in `.env`

### 5. Create Auth Hooks
- [ ] Create `src/hooks/use-session.ts` for client-side session access
- [ ] Create server-side `getSession()` utility
- [ ] Create `getCurrentUser()` utility

### 6. Configure Environment Variables
- [ ] Add `AUTH_SECRET` to `.env`
- [ ] Add `AUTH_URL` to `.env`
- [ ] Document in `.env.example`

---

## Verification Criteria

- [ ] NextAuth installed
- [ ] Credentials provider configured
- [ ] Login API route working
- [ ] Session callback returns user data
- [ ] JWT tokens generated correctly
- [ ] TypeScript types resolved

---

## Notes

- next-auth v5 (beta) for Next.js 16 compatibility
- Use `AUTH_SECRET` for token signing (generate with `openssl rand -base64 32`)
- JWT strategy works better with Vercel edge than database sessions
