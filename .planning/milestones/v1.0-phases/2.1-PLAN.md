# Phase 2.1: Set up PostgreSQL (local + Vercel Postgres)

**Phase:** 2
**Plan:** 2.1
**Status:** Ready to Execute

---

## Objective

Set up PostgreSQL database for development and configure for Vercel production.

---

## Tasks

### 1. Install Prisma
- [ ] Run `npm install prisma @prisma/client`
- [ ] Run `npx prisma init`
- [ ] Verify `prisma/` directory created with `schema.prisma`

### 2. Configure Prisma for PostgreSQL
- [ ] Update `prisma/schema.prisma` datasource provider to `postgresql`
- [ ] Create `.env` file with `DATABASE_URL`
- [ ] Add `.env` to `.gitignore` (already there)

### 3. Set Up Local PostgreSQL
**Option A: Docker (Recommended)**
- [ ] Create `docker-compose.yml` with PostgreSQL service
- [ ] Run `docker-compose up -d`
- [ ] Verify database is running

**Option B: Direct Install**
- [ ] Document manual PostgreSQL installation
- [ ] Create database `queue_automation`

### 4. Configure Vercel Postgres
- [ ] Document Vercel Postgres setup in README
- [ ] Add `vercel.json` with database configuration
- [ ] Create `.env.example` with placeholder

### 5. Test Database Connection
- [ ] Create simple test script
- [ ] Verify connection works
- [ ] Document connection troubleshooting

---

## Verification Criteria

- [ ] Prisma installed and initialized
- [ ] `schema.prisma` configured for PostgreSQL
- [ ] Local database running
- [ ] `DATABASE_URL` in `.env`
- [ ] Connection test passes
- [ ] Vercel setup documented

- PostgreSQL chosen for reliability and Vercel integration
