# Phase 1.3: Set up ESLint, Prettier, Husky

**Phase:** 1
**Plan:** 1.3
**Status:** Ready to Execute

---

## Objective

Configure code quality tools: ESLint, Prettier, and pre-commit hooks with Husky.

---

## Tasks

### 1. Install Prettier
- [ ] Run `npm install -D prettier eslint-config-prettier`
- [ ] Create `.prettierrc` with config:
  ```json
  {
    "semi": false,
    "singleQuote": true,
    "tabWidth": 2,
    "trailingComma": "es5",
    "printWidth": 100
  }
  ```
- [ ] Create `.prettierignore` (exclude `.planning/`, `node_modules/`, etc.)

### 2. Update ESLint Config
- [ ] Update `.eslintrc.json` to extend `prettier`
- [ ] Add TypeScript-specific rules
- [ ] Configure React hooks rules

### 3. Install Husky and lint-staged
- [ ] Run `npm install -D husky lint-staged`
- [ ] Run `npx husky init`
- [ ] Create `.husky/pre-commit` with:
  ```bash
  npx lint-staged
  ```

### 4. Configure lint-staged
- [ ] Add to `package.json`:
  ```json
  "lint-staged": {
    "*.{ts,tsx}": ["eslint --fix", "prettier --write"],
    "*.{json,md}": ["prettier --write"]
  }
  ```

### 5. Test Pre-commit Hook
- [ ] Make a small code change
- [ ] Stage and commit
- [ ] Verify lint-staged runs automatically

### 6. Add npm Scripts
- [ ] Add `format`: `prettier --write .`
- [ ] Add `lint`: `eslint . --ext .ts,.tsx`
- [ ] Add `type-check`: `tsc --noEmit`

---

## Verification Criteria

- [ ] `.prettierrc` exists with correct config
- [ ] ESLint runs without errors: `npm run lint`
- [ ] Prettier formats files: `npm run format`
- [ ] Husky pre-commit hook installed
- [ ] lint-staged runs on git commit
- [ ] Commit blocked if linting fails

---

## Notes

- Next.js 15 includes ESLint by default - extend that config
- Prettier handles formatting, ESLint handles code quality
- Husky hooks are git hooks - run on every commit
