# Phase 1.2: Configure Tailwind CSS + shadcn/ui

**Phase:** 1
**Plan:** 1.2
**Status:** Ready to Execute

---

## Objective

Install and configure shadcn/ui component library with Tailwind CSS.

---

## Tasks

### 1. Initialize shadcn/ui
- [ ] Run `npx shadcn-ui@latest init`
- [ ] Select options:
  - Style: Default
  - Base color: Slate (or custom brand color)
  - CSS variables: Yes
  - Tailwind config: Use existing

### 2. Install Base Components
- [ ] Run `npx shadcn-ui@latest add button card input label alert-dialog skeleton table badge tabs`
- [ ] Verify components installed in `src/components/ui/`

### 3. Install next-themes for Dark Mode
- [ ] Run `npm install next-themes`
- [ ] Create theme provider component
- [ ] Add dark mode toggle to layout

### 4. Configure Custom Theme
- [ ] Update `tailwind.config.ts` with custom colors
- [ ] Add custom font (Inter or Geist)
- [ ] Configure container queries if needed

### 5. Create Base Layout Components
- [ ] Create `src/components/layout/header.tsx`
- [ ] Create `src/components/layout/footer.tsx`
- [ ] Update `src/app/layout.tsx` with providers

---

## Verification Criteria

- [ ] shadcn/ui initialized (check `components.json`)
- [ ] 8+ UI components in `src/components/ui/`
- [ ] Dark mode toggle working
- [ ] Button component renders correctly
- [ ] Card component renders correctly
- [ ] Tailwind classes apply correctly

---

## Notes

- shadcn/ui copies code to project - fully customizable
- Components use Radix UI primitives for accessibility
- Theme colors defined in CSS variables for easy dark mode
