## Phase 2: Complete WhatsApp Integration

**Gaps Closed:**
- WA-01 through WA-05 (not implemented per spec)

**Tasks:**
1. Create `src/lib/whatsapp/` directory structure
2. Implement `evolution-api.ts` — API client wrapper
3. Implement `templates.ts` — message templates for all 5 notification types
4. Implement `notifications.ts` — orchestration layer
5. Create `src/types/whatsapp.ts` — type definitions
6. Create `src/app/api/whatsapp/send/route.ts` — send endpoint
7. Wire notification triggers to queue events:
   - Queue join → WA-01 (confirmation)
   - Position 3 → WA-02 (turn approaching)
   - Call next → WA-03 (turn called)
   - Mark served → WA-04 (completion)
   - 1hr after served → WA-05 (feedback request)
8. Implement retry logic (max 3 attempts)
9. Log delivery failures to database

**Success Criteria:**
- All 5 notification types trigger correctly
- Templates render with correct data
- Failed messages retried 3x then logged
- Integration test verifies end-to-end flow
